# 2.4 — Objetos de Banco de Dados

> Além das tabelas, um banco de dados relacional contém outros **objetos** que organizam, aceleram e automatizam o trabalho com os dados.

---

## Tabela (*Table*)

- O objeto **principal** — armazena os dados em linhas e colunas.
- Possui esquema fixo, chave primária e, opcionalmente, chaves estrangeiras.

---

## Exibição / Visão (*View*)

- Uma **consulta salva** que se comporta como uma tabela virtual.
- **Não armazena dados** — apresenta o resultado de um `SELECT` sempre atualizado.
- Usos: simplificar consultas complexas, **expor só parte** dos dados (segurança), padronizar relatórios.

```sql
CREATE VIEW ClientesRecife AS
SELECT Nome, Email FROM Cliente WHERE Cidade = 'Recife';
```

---

## Procedimento Armazenado (*Stored Procedure*)

- Um **bloco de código SQL** salvo no banco, executado sob demanda.
- Pode receber **parâmetros** e conter lógica (vários comandos, condições).
- Usos: encapsular regras de negócio, reaproveitar código, reduzir tráfego de rede.

```sql
CREATE PROCEDURE InserirCliente @Nome VARCHAR(100), @Cidade VARCHAR(50)
AS
INSERT INTO Cliente (Nome, Cidade) VALUES (@Nome, @Cidade);
```

---

## Índice (*Index*)

- Estrutura que **acelera a busca** de linhas por uma ou mais colunas.
- Funciona como o índice de um livro: evita varrer a tabela inteira.
- **Vantagem:** consultas mais rápidas. **Custo:** ocupa espaço e torna a escrita um pouco mais lenta (o índice precisa ser atualizado).

| Tipo de índice | Característica |
|----------------|----------------|
| **Clustered** | Define a **ordem física** das linhas na tabela; só pode haver **um** por tabela. |
| **Non-clustered** | Estrutura separada que aponta para as linhas; pode haver **vários** por tabela. |

---

## Resumo dos Objetos

| Objeto | Armazena dados? | Função principal |
|--------|-----------------|------------------|
| **Tabela** | Sim | Guardar os dados |
| **View** | Não | Consulta salva / tabela virtual |
| **Stored Procedure** | Não | Bloco de código SQL reutilizável |
| **Índice** | Não (referência) | Acelerar buscas |

---

## Pontos-Chave para o Exame

- ✅ **View** = consulta salva; **não armazena dados**.
- ✅ **Stored procedure** = bloco de código SQL reutilizável, aceita parâmetros.
- ✅ **Índice** acelera leitura, mas custa espaço e desempenho de escrita.
- ✅ Índice **clustered** define a ordem física e há **só um** por tabela.
