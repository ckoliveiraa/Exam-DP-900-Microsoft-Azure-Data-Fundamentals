# 2.3 — SQL: Comandos Comuns

> **SQL** (*Structured Query Language*) é a linguagem padrão dos bancos relacionais. Existe desde os anos 70 e, até hoje, é o "idioma" que todo SGBD relacional entende — do SQL Server ao PostgreSQL, passando pelo Oracle e MySQL.

Para organizar tantos comandos diferentes, o SQL é dividido em **famílias**, cada uma com um propósito bem definido. Saber a qual família um comando pertence é, possivelmente, o conteúdo mais cobrado deste módulo na prova.

---

## As Três Famílias de Comandos

| Família | Nome completo | Para que serve | Comandos principais |
|---------|---------------|----------------|---------------------|
| **DDL** | *Data Definition Language* | Define a **estrutura** do banco (objetos) | `CREATE`, `ALTER`, `DROP` |
| **DML** | *Data Manipulation Language* | Manipula os **dados** dentro das tabelas | `SELECT`, `INSERT`, `UPDATE`, `DELETE` |
| **DCL** | *Data Control Language* | Controla **permissões** de acesso | `GRANT`, `REVOKE`, `DENY` |

> 💡 Em alguns materiais, o `SELECT` aparece em uma quarta família, a **DQL** (*Data Query Language*). Para o DP-900, basta saber que `SELECT` é o comando que **consulta** dados — não importa se é classificado como DML ou DQL.

---

## DDL — Definindo a Estrutura

Comandos DDL alteram o **esqueleto** do banco: criam tabelas, mudam suas colunas, removem objetos. Eles **não mexem nos dados** diretamente — mexem nas "caixas" que os guardam.

```sql
-- Cria uma nova tabela
CREATE TABLE Cliente (
    ClienteID INT PRIMARY KEY,
    Nome      VARCHAR(100),
    Cidade    VARCHAR(50)
);

-- Adiciona uma coluna à tabela existente
ALTER TABLE Cliente ADD Email VARCHAR(100);

-- Exclui a tabela inteira (estrutura + dados)
DROP TABLE Cliente;
```

| Comando | O que faz |
|---------|-----------|
| `CREATE` | Cria um objeto (tabela, view, índice, procedure...) |
| `ALTER`  | Modifica a estrutura de um objeto que já existe |
| `DROP`   | Exclui um objeto do banco por completo |

---

## DML — Manipulando os Dados

Se DDL cuida das "caixas", DML cuida do **conteúdo** delas: ler, inserir, atualizar e remover linhas. É a família que você mais usa no dia a dia.

```sql
-- Consultar dados
SELECT Nome, Cidade FROM Cliente WHERE Cidade = 'Recife';

-- Inserir uma nova linha
INSERT INTO Cliente (ClienteID, Nome, Cidade)
VALUES (1, 'Ana Souza', 'Recife');

-- Atualizar linhas existentes
UPDATE Cliente SET Cidade = 'Olinda' WHERE ClienteID = 1;

-- Remover linhas
DELETE FROM Cliente WHERE ClienteID = 1;
```

| Comando | O que faz |
|---------|-----------|
| `SELECT` | Lê (consulta) dados de uma ou mais tabelas |
| `INSERT` | Adiciona novas linhas |
| `UPDATE` | Altera os valores de linhas existentes |
| `DELETE` | Remove linhas |

> ⚠️ **Cuidado clássico:** `UPDATE` e `DELETE` **sem cláusula `WHERE`** afetam **todas as linhas** da tabela. Esquecer o `WHERE` é uma das maiores fontes de incidentes em produção — e uma pegadinha frequente na prova.

---

## DCL — Controlando Permissões

DCL define **quem pode fazer o quê** dentro do banco. É a camada de segurança que decide se um usuário pode ler uma tabela, alterá-la ou nem mesmo enxergá-la.

```sql
-- Concede permissão de leitura
GRANT SELECT ON Cliente TO usuario_relatorios;

-- Remove uma permissão previamente concedida
REVOKE SELECT ON Cliente FROM usuario_relatorios;
```

| Comando | O que faz |
|---------|-----------|
| `GRANT`  | Concede uma permissão a um usuário ou papel |
| `REVOKE` | Remove uma permissão previamente concedida |
| `DENY`   | Nega explicitamente uma permissão (sobrepõe um `GRANT` herdado) |

---

## Dialetos do SQL

O SQL tem um **padrão ANSI**, mas cada fornecedor adiciona suas próprias extensões. Os comandos básicos (`SELECT`, `INSERT`, `UPDATE`, `DELETE`) funcionam em todos — as diferenças aparecem em recursos avançados como funções, controle de fluxo e tipos específicos.

| Dialeto | Usado em |
|---------|----------|
| **T-SQL** (*Transact-SQL*) | **SQL Server e Azure SQL** |
| **PL/pgSQL** | PostgreSQL |
| **PL/SQL** | Oracle |

> Para o DP-900, lembre-se: **T-SQL é o dialeto da Microsoft** — tudo que envolver SQL Server ou Azure SQL na prova usa T-SQL.

---

## Pontos-Chave para o Exame

- ✅ **DDL** define a estrutura: `CREATE`, `ALTER`, `DROP`.
- ✅ **DML** manipula os dados: `SELECT`, `INSERT`, `UPDATE`, `DELETE`.
- ✅ **DCL** controla permissões: `GRANT`, `REVOKE`, `DENY`.
- ✅ `UPDATE` e `DELETE` **sem `WHERE`** afetam a tabela inteira.
- ✅ **T-SQL** é o dialeto usado pelo **SQL Server** e pelo **Azure SQL**.
- ✅ Cuidado para não confundir: `DROP` remove a **tabela**; `DELETE` remove **linhas**.
