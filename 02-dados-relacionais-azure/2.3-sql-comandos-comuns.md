# 2.3 — SQL: Comandos Comuns

> **SQL** (*Structured Query Language*) é a linguagem padrão para consultar e gerenciar bancos de dados relacionais. Seus comandos se dividem em **três famílias**: DDL, DML e DCL.

---

## As Três Famílias de Comandos

| Família | Nome | Para que serve | Comandos |
|---------|------|----------------|----------|
| **DDL** | *Data Definition Language* | Definir a **estrutura** do banco | `CREATE`, `ALTER`, `DROP` |
| **DML** | *Data Manipulation Language* | Manipular os **dados** | `SELECT`, `INSERT`, `UPDATE`, `DELETE` |
| **DCL** | *Data Control Language* | Controlar **permissões** | `GRANT`, `REVOKE`, `DENY` |

> Alguns materiais classificam o `SELECT` como **DQL** (*Data Query Language*). No exame, basta saber que ele **consulta** dados.

---

## DDL — Definir a Estrutura

```sql
CREATE TABLE Cliente (
    ClienteID INT PRIMARY KEY,
    Nome      VARCHAR(100),
    Cidade    VARCHAR(50)
);

ALTER TABLE Cliente ADD Email VARCHAR(100);

DROP TABLE Cliente;
```

| Comando | Ação |
|---------|------|
| `CREATE` | Cria um objeto (tabela, view, índice) |
| `ALTER` | Modifica a estrutura de um objeto existente |
| `DROP` | Exclui um objeto |

---

## DML — Manipular os Dados

```sql
-- Consultar
SELECT Nome, Cidade FROM Cliente WHERE Cidade = 'Recife';

-- Inserir
INSERT INTO Cliente (ClienteID, Nome, Cidade) VALUES (1, 'Ana Souza', 'Recife');

-- Atualizar
UPDATE Cliente SET Cidade = 'Olinda' WHERE ClienteID = 1;

-- Excluir
DELETE FROM Cliente WHERE ClienteID = 1;
```

| Comando | Ação |
|---------|------|
| `SELECT` | Lê/consulta dados |
| `INSERT` | Adiciona novas linhas |
| `UPDATE` | Altera linhas existentes |
| `DELETE` | Remove linhas |

> ⚠️ `UPDATE` e `DELETE` **sem `WHERE`** afetam **todas** as linhas da tabela.

---

## DCL — Controlar Permissões

```sql
GRANT SELECT ON Cliente TO usuario_relatorios;
REVOKE SELECT ON Cliente FROM usuario_relatorios;
```

| Comando | Ação |
|---------|------|
| `GRANT` | Concede uma permissão |
| `REVOKE` | Remove uma permissão concedida |
| `DENY` | Nega explicitamente uma permissão |

---

## Dialetos do SQL

O SQL tem um padrão ANSI, mas cada fornecedor adiciona extensões:

| Dialeto | Usado em |
|---------|----------|
| **T-SQL** (*Transact-SQL*) | SQL Server, Azure SQL |
| **PL/pgSQL** | PostgreSQL |
| **PL/SQL** | Oracle |

> Os comandos básicos (`SELECT`, `INSERT`...) funcionam em todos; as diferenças estão em recursos avançados.

---

## Pontos-Chave para o Exame

- ✅ **DDL** define estrutura (`CREATE`, `ALTER`, `DROP`).
- ✅ **DML** manipula dados (`SELECT`, `INSERT`, `UPDATE`, `DELETE`).
- ✅ **DCL** controla permissões (`GRANT`, `REVOKE`, `DENY`).
- ✅ `UPDATE`/`DELETE` sem `WHERE` afetam **toda** a tabela.
- ✅ **T-SQL** é o dialeto do SQL Server e do Azure SQL.
