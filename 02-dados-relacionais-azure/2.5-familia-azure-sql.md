# 2.5 — Família Azure SQL

> A **família Azure SQL** reúne os serviços do Azure para bancos de dados relacionais baseados no **Microsoft SQL Server**. A diferença entre eles está no equilíbrio entre **controle** e **gestão automática**.

---

## IaaS × PaaS

| Modelo | Quem gerencia o quê | Exemplo na família |
|--------|---------------------|--------------------|
| **IaaS** (*Infrastructure as a Service*) | Você gerencia o SO e o SQL Server; a Azure fornece a VM. | SQL Server em VMs do Azure |
| **PaaS** (*Platform as a Service*) | A Azure gerencia infraestrutura, SO e maior parte do banco. | Azure SQL Database, SQL Managed Instance |

> Quanto mais **PaaS**, **menos** trabalho de administração — e menos controle sobre o ambiente.

---

## Os Três Serviços

### Azure SQL Database (PaaS)

- Banco de dados **totalmente gerenciado** na nuvem.
- A Azure cuida de patches, backups, alta disponibilidade e atualizações.
- **Não** dá acesso ao sistema operacional nem à instância completa do SQL Server.
- Ideal para **aplicações modernas** criadas para a nuvem (*cloud-native*).
- Opções de implantação: **banco único** ou **pool elástico** (vários bancos compartilham recursos).

### Azure SQL Managed Instance (PaaS)

- Banco gerenciado, mas com **quase 100% de compatibilidade** com o SQL Server local.
- Oferece recursos de **instância** (ex.: SQL Agent, consultas entre bancos) que o SQL Database não tem.
- Ideal para **migrar bancos locais** para a nuvem com o **mínimo de alterações** ("lift and shift").

### SQL Server em VMs do Azure (IaaS)

- O SQL Server instalado em uma **máquina virtual** do Azure.
- **Controle total** sobre o SO e a instância do SQL Server.
- Você é responsável por patches, backups e atualizações.
- Ideal quando a aplicação exige **acesso ao SO** ou recursos não disponíveis no PaaS.

---

## Como Escolher

| Necessidade | Serviço indicado |
|-------------|------------------|
| Aplicação nova, nuvem, mínimo de gestão | **Azure SQL Database** |
| Migrar banco local com poucas mudanças | **SQL Managed Instance** |
| Controle total do SO e da instância | **SQL Server em VM** |

---

## Pontos-Chave para o Exame

- ✅ **SQL Database** = PaaS, totalmente gerenciado, para apps cloud-native.
- ✅ **SQL Managed Instance** = PaaS com alta compatibilidade — ideal para **migração**.
- ✅ **SQL Server em VM** = IaaS, controle total, **mais** responsabilidade do cliente.
- ✅ Mais **PaaS** = menos administração; mais **IaaS** = mais controle.
- ✅ **Pool elástico** permite vários bancos do SQL Database compartilharem recursos.
