# 2.6 — Bancos Open-Source no Azure

> Além do SQL Server, o Azure oferece serviços **totalmente gerenciados** para os bancos de dados relacionais **open-source** mais populares — sem que você precise instalar ou administrar servidores.

---

## Os Serviços Gerenciados

| Serviço | Banco open-source | Observação |
|---------|-------------------|------------|
| **Azure Database for PostgreSQL** | PostgreSQL | Banco objeto-relacional avançado |
| **Azure Database for MySQL** | MySQL | Muito usado em aplicações web |
| **Azure Database for MariaDB** | MariaDB | Derivado do MySQL — *em processo de desativação* |

> ⚠️ A Microsoft **descontinuou** o Azure Database for MariaDB. O exame ainda pode mencioná-lo, mas o foco atual é **PostgreSQL** e **MySQL**.

---

## O que "Gerenciado" Significa

Ao usar esses serviços (modelo **PaaS**), o Azure cuida automaticamente de:

- **Provisionamento** e configuração do servidor.
- **Patches** e atualizações do banco.
- **Backups automáticos** e restauração para um ponto no tempo.
- **Alta disponibilidade** integrada.
- **Segurança** (criptografia, firewall) e **monitoramento**.

Você foca apenas nos **dados e nas consultas** — não na infraestrutura.

---

## Por que Usar Bancos Open-Source no Azure

- **Compatibilidade total** com o banco open-source original — aplicações existentes funcionam sem reescrita.
- Facilita **migrar** sistemas que já usam MySQL ou PostgreSQL para a nuvem.
- Evita o trabalho de instalar, atualizar e proteger servidores manualmente.
- Combina o **ecossistema open-source** com a **escala e a confiabilidade** do Azure.

---

## Azure Database for PostgreSQL — Opção Flexível

O PostgreSQL no Azure é oferecido na opção de implantação **Flexible Server**, que dá mais controle sobre configuração, janelas de manutenção e otimização de custos — mantendo o serviço gerenciado.

---

## Pontos-Chave para o Exame

- ✅ O Azure tem serviços gerenciados para **MySQL**, **PostgreSQL** e **MariaDB**.
- ✅ São **PaaS**: o Azure cuida de patches, backups, HA e segurança.
- ✅ Mantêm **compatibilidade** com o banco original → facilitam a **migração**.
- ✅ O **MariaDB** está sendo **descontinuado** — prefira PostgreSQL/MySQL.
