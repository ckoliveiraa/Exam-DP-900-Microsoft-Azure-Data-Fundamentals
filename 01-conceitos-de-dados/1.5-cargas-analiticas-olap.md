# 1.5 — Cargas de Trabalho Analíticas (OLAP)

> Uma **carga de trabalho analítica** (*OLAP — Online Analytical Processing*) processa **grandes volumes de dados históricos** para gerar resumos, identificar tendências e **apoiar a tomada de decisão**.

---

## Características do OLAP

- Consultas **complexas** que leem e agregam **muitos registros** de uma vez.
- Foco em **agregações**: somas, médias, contagens, máximos, tendências.
- Dados predominantemente em **somente leitura** — pouca ou nenhuma escrita do usuário final.
- Otimizado para **leitura**, não para escrita rápida.
- Trabalha com dados **históricos** (vários anos), não apenas o estado atual.
- Dados frequentemente **desnormalizados** (esquema estrela) para acelerar consultas.
- Volumes grandes → armazenamento orientado a **coluna** (ex.: Parquet).

## Exemplos de Sistemas OLAP

- Relatórios de vendas por região, produto e período.
- Painéis (dashboards) de indicadores de negócio (KPIs).
- Previsão de demanda a partir de séries históricas.
- Análise de comportamento de clientes (segmentação).

---

## O Caminho dos Dados: ETL / ELT

Dados transacionais (OLTP) não são analisados diretamente — eles são **movidos e preparados** antes de chegar a um repositório analítico.

| Etapa | O que faz |
|-------|-----------|
| **Extract** (Extração) | Lê os dados das fontes (bancos OLTP, arquivos, APIs). |
| **Transform** (Transformação) | Limpa, padroniza, agrega e reformata os dados. |
| **Load** (Carga) | Grava os dados no repositório analítico. |

- **ETL** — transforma **antes** de carregar (destino: data warehouse).
- **ELT** — carrega os dados brutos **primeiro** e transforma depois (destino: data lake).

O destino final é um **data warehouse**, **data lake** ou **lakehouse**, onde as consultas OLAP são executadas (detalhado no Módulo 4).

---

## Esquema Estrela (desnormalização)

Em OLAP, os dados são organizados em **tabelas de fatos** (medidas numéricas — ex.: valor da venda) cercadas por **tabelas de dimensão** (contexto — produto, tempo, local). Esse formato **desnormalizado** reduz o número de junções e acelera as consultas analíticas.

---

## OLTP × OLAP

| Aspecto | OLTP (transacional) | OLAP (analítico) |
|---------|---------------------|------------------|
| Objetivo | Registrar operações do dia a dia | Analisar dados e apoiar decisões |
| Operações | Muitas e pequenas (leitura/escrita) | Poucas e complexas (leitura) |
| Volume por consulta | Registros individuais | Milhões de registros agregados |
| Estado dos dados | Atuais | Históricos |
| Modelagem | Normalizada | Desnormalizada (esquema estrela) |
| Otimizado para | Escrita rápida e concorrência | Leitura e agregação |
| Usuários | Operacionais (clientes, atendentes) | Analistas e gestores |

---

## Pontos-Chave para o Exame

- ✅ **OLAP = análise de grandes volumes** de dados **históricos**.
- ✅ Otimizado para **leitura e agregação**, não para escrita.
- ✅ Dados costumam ser **desnormalizados** (esquema estrela: fatos + dimensões).
- ✅ Dados chegam ao repositório analítico via **ETL/ELT**.
- ✅ **ETL** transforma antes de carregar; **ELT** carrega bruto e transforma depois.
- ✅ Saiba **diferenciar OLTP × OLAP** — questão clássica do exame.
