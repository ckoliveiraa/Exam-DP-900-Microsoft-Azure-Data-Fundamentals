# 1.3 — Opções de Armazenamento de Dados

> A escolha de **onde** armazenar os dados depende do **tipo de dado** e da **carga de trabalho**. As três grandes opções são: **bancos relacionais**, **bancos não relacionais** e **armazenamento de arquivos**.

---

## Bancos de Dados Relacionais

Modelo de **tabelas** com esquema fixo, relacionadas entre si por chaves.

- Cada **tabela** representa um tipo de entidade (clientes, pedidos, produtos).
- Cada **linha** é uma instância; cada **coluna** tem um tipo de dado definido.
- **Chave primária** identifica unicamente cada linha; **chave estrangeira** liga uma tabela a outra.
- Garantem integridade pelas propriedades **ACID** (atomicidade, consistência, isolamento, durabilidade).
- Consultados com **SQL**.
- Ideais para dados estruturados que exigem **consistência** e **relacionamentos**.
- Exemplos: Azure SQL Database, SQL Server, MySQL, PostgreSQL.

---

## Bancos de Dados Não Relacionais (NoSQL)

Não usam o modelo de tabelas rígidas. Trocam parte da consistência estrita por **flexibilidade de esquema** e **escala horizontal**. Há **quatro tipos**:

### 1. Documento

- Cada item é um **documento JSON** completo (uma entidade inteira).
- Esquema flexível — documentos podem ter campos diferentes.
- Exemplo no Azure: **Cosmos DB — API for NoSQL**.

### 2. Chave-valor

- O banco mais simples: cada item é uma **chave única** associada a um **valor**.
- Acesso muito rápido pela chave; o banco não "enxerga" o conteúdo do valor.
- Exemplo no Azure: **Azure Table Storage**.

### 3. Família de colunas (*column-family*)

- Linhas identificadas por chave, com colunas agrupadas em **famílias** lidas em conjunto.
- Boa leitura em grande escala.
- Exemplo no Azure: **Cosmos DB — API for Cassandra**.

### 4. Grafo

- Armazena **nós** (entidades) e **arestas** (relações entre entidades).
- Ideal para dados muito conectados (redes sociais, recomendações, fraude).
- Exemplo no Azure: **Cosmos DB — API for Apache Gremlin**.

---

## Armazenamento de Arquivos

Guarda dados como **arquivos/objetos**, sem impor esquema.

- Ideal para dados **não estruturados** (mídia, documentos, backups, logs).
- Base dos **data lakes** (repositórios de dados brutos em larga escala).
- Opções no Azure:
  - **Azure Blob Storage** — objetos/arquivos não estruturados em grande escala.
  - **Azure Files** — compartilhamentos de arquivos acessados via protocolo SMB.

---

## Comparativo Geral

| | Relacional | Não relacional | Arquivos |
|---|---|---|---|
| Esquema | Fixo (*schema-on-write*) | Flexível | Nenhum |
| Tipo de dado | Estruturado | Semiestruturado | Não estruturado |
| Linguagem de consulta | SQL | API / linguagem própria | Processamento de arquivos |
| Relacionamentos/integridade | Fortes (ACID) | Limitados | Não se aplica |
| Escala | Vertical (principalmente) | Horizontal | Horizontal |

## Como Escolher

| Necessidade | Opção indicada |
|-------------|----------------|
| Dados tabulares, integridade e relacionamentos | Banco **relacional** |
| Esquema flexível, alta escala, documentos JSON | Banco **não relacional** |
| Dados muito conectados (redes, recomendações) | NoSQL de **grafo** |
| Mídia, documentos, arquivos brutos, data lake | **Armazenamento de arquivos** |

---

## Pontos-Chave para o Exame

- ✅ Três grandes opções: **relacional**, **não relacional** e **arquivos**.
- ✅ NoSQL tem **4 tipos**: documento, chave-valor, família de colunas e grafo.
- ✅ Relacional → esquema fixo, SQL e integridade ACID.
- ✅ **Grafo** → dados muito conectados; **chave-valor** → acesso simples e rápido por chave.
- ✅ Arquivos/Blob → dados não estruturados e base de **data lakes**.
