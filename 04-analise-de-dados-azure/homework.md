## Parte 2 — Desafios

### Desafio 1 — Desenhe o fluxo de análise

Escolha um cenário (vendas de uma rede de lojas, sensores de uma fábrica, uso de um app). Descreva as **5 etapas** do fluxo de análise para esse caso:

1. De onde vêm os dados (ingestão).
2. O que precisa ser transformado.
3. Onde serão armazenados.
4. Como serão modelados.
5. Como serão visualizados.

> **Entrega:** um diagrama ou lista com as 5 etapas descritas.

---

### Desafio 2 — Warehouse, lake ou lakehouse?

Para cada necessidade, escolha o repositório analítico e justifique:

1. Relatórios de BI sobre dados de vendas já estruturados e modelados.
2. Armazenar dados brutos de vários formatos (logs, JSON, imagens) ao menor custo.
3. Uma solução única que sirva tanto para dados brutos quanto para BI de alto desempenho.

> **Entrega:** as três escolhas, com justificativa.

---

### Desafio 3 — Escolha o serviço do Azure

Para cada tarefa, indique o serviço de análise mais adequado e explique:

1. Criar um pipeline que ingere e transforma dados de várias fontes.
2. Treinar um modelo de machine learning sobre grande volume de dados com Spark.
3. Adotar uma plataforma SaaS única que cubra ingestão, lake, warehouse e Power BI.

> **Entrega:** os três serviços escolhidos, com justificativa.

---

### Desafio 4 — Batch ou streaming?

Para cada situação, decida entre processamento em lote ou em fluxo e justifique:

1. Relatório consolidado de faturamento gerado toda madrugada.
2. Painel que mostra, em tempo real, a temperatura de equipamentos críticos.
3. Detecção imediata de transações fraudulentas em um cartão de crédito.

> **Entrega:** as três decisões, com justificativa baseada em latência.

---

### Desafio 5 — Planeje um painel no Power BI

Imagine um painel de vendas para a diretoria. Defina:

- Quais **indicadores** ele mostraria.
- Qual **visualização** usar para cada um (e por quê).
- A diferença entre entregar isso como **relatório** ou como **painel**.

> **Entrega:** o esboço do painel com os visuais escolhidos e a justificativa.
