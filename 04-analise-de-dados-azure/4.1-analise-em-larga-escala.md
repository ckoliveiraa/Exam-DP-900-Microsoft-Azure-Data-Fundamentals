# 4.1 — Elementos da Análise em Larga Escala

> A **análise em larga escala** é o processo de transformar grandes volumes de dados brutos em **informação útil** para a tomada de decisão. Ela segue um fluxo com etapas bem definidas.

---

## O Fluxo da Análise de Dados

| Etapa | O que faz |
|-------|-----------|
| **1. Ingestão** | Captura os dados das fontes (bancos, arquivos, dispositivos, APIs). |
| **2. Processamento / Transformação** | Limpa, padroniza, combina e agrega os dados (ETL/ELT). |
| **3. Armazenamento analítico** | Grava os dados preparados em um data warehouse, data lake ou lakehouse. |
| **4. Modelagem** | Organiza os dados em modelos analíticos (ex.: esquema estrela). |
| **5. Visualização** | Apresenta os resultados em relatórios e painéis. |

---

## Ingestão de Dados

A **ingestão** traz os dados das fontes para a plataforma de análise. Pode ser feita de duas formas:

- **Lote (batch)** — dados coletados e processados em grupos, em intervalos definidos.
- **Fluxo (streaming)** — dados processados continuamente, à medida que chegam.

> Os pipelines de ingestão e transformação são criados por ferramentas como o **Azure Data Factory** e os **pipelines do Microsoft Fabric**.

---

## Processamento: ETL × ELT

| | ETL | ELT |
|---|---|---|
| Ordem | Extrair → **Transformar** → Carregar | Extrair → Carregar → **Transformar** |
| Transformação | Antes de gravar no destino | Depois de gravar, já no destino |
| Destino típico | Data warehouse | Data lake / lakehouse |

---

## Pipelines de Dados

Um **pipeline** é uma sequência automatizada de atividades que move e transforma os dados de ponta a ponta. Ele orquestra a ingestão, a transformação e a carga — de forma agendada ou disparada por eventos.

---

## Pontos-Chave para o Exame

- ✅ A análise em larga escala segue o fluxo: **ingestão → processamento → armazenamento → modelagem → visualização**.
- ✅ A **ingestão** pode ser em **lote (batch)** ou em **fluxo (streaming)**.
- ✅ **ETL** transforma antes de carregar; **ELT** carrega e depois transforma.
- ✅ **Pipelines** (ex.: Azure Data Factory) automatizam e orquestram esse fluxo.
