# 3.5 — APIs do Cosmos DB

> O Cosmos DB é **multimodelo**: ele suporta diferentes **APIs**, cada uma compatível com um modelo de dados e uma tecnologia conhecida. A API escolhida define **como** você armazena e consulta os dados.

---

## Por que Várias APIs

A API permite que aplicações já escritas para outros bancos NoSQL usem o Cosmos DB **com pouca ou nenhuma alteração** de código — aproveitando a escala global e o gerenciamento do serviço. A API é escolhida na **criação da conta** e não pode ser trocada depois.

---

## As APIs Disponíveis

| API | Modelo de dados | Quando escolher |
|-----|-----------------|-----------------|
| **API for NoSQL** | Documentos JSON | É a **API nativa** do Cosmos DB; recomendada para projetos novos. Consulta com sintaxe semelhante a SQL. |
| **API for MongoDB** | Documentos (BSON) | Migrar ou reaproveitar aplicações que já usam **MongoDB**. |
| **API for Apache Cassandra** | Família de colunas | Migrar ou reaproveitar aplicações que usam **Cassandra**. |
| **API for Apache Gremlin** | **Grafo** (nós e arestas) | Dados altamente conectados — redes sociais, recomendações, detecção de fraude. |
| **API for Table** | Chave-valor | Aplicações que usam o **Azure Table Storage** e precisam de mais recursos e distribuição global. |

---

## Como Escolher a API

| Situação | API indicada |
|----------|--------------|
| Projeto novo, sem dependência de outro banco | **NoSQL** (nativa) |
| App existente em MongoDB | **MongoDB** |
| App existente em Cassandra | **Cassandra** |
| Dados muito conectados (relações entre entidades) | **Gremlin** (grafo) |
| App de Table Storage que precisa de escala global | **Table** |

> 💡 Regra prática: para um **projeto novo**, use a **API for NoSQL**. As outras existem principalmente para **compatibilidade/migração** de aplicações já feitas para aquela tecnologia.

---

## Pontos-Chave para o Exame

- ✅ O Cosmos DB é **multimodelo** e oferece **5 APIs**: NoSQL, MongoDB, Cassandra, Gremlin e Table.
- ✅ A **API for NoSQL** é a nativa — recomendada para **projetos novos**.
- ✅ MongoDB, Cassandra e Table existem sobretudo para **compatibilidade/migração**.
- ✅ A **API for Gremlin** é para dados de **grafo** (entidades muito conectadas).
- ✅ A API é definida na **criação da conta** e não muda depois.
