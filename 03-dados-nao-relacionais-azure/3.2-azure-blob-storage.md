# 3.2 — Azure Blob Storage

> O **Azure Blob Storage** é o serviço para armazenar grandes volumes de dados **não estruturados** como **objetos** (*blobs* — *Binary Large Objects*): imagens, vídeos, documentos, backups, logs.

---

## Estrutura: Conta → Contêiner → Blob

| Nível | O que é |
|-------|---------|
| **Conta de armazenamento** | O contêiner de nível superior dos serviços. |
| **Contêiner** | Agrupa blobs — equivalente a uma "pasta" de nível raiz. |
| **Blob** | O objeto/arquivo em si. |

> O Blob Storage tem um espaço de nomes "plano", mas é possível **simular pastas** usando `/` no nome do blob.

---

## Tipos de Blob

| Tipo | Uso |
|------|-----|
| **Block blob** | Arquivos comuns (documentos, mídia) — o tipo mais usado. |
| **Append blob** | Otimizado para **acréscimos** ao final, como arquivos de log. |
| **Page blob** | Acesso aleatório; base dos **discos de VMs** do Azure. |

---

## Camadas de Acesso (*Access Tiers*)

Permitem **otimizar custo** conforme a frequência de uso dos dados:

| Camada | Dados | Custo de armazenamento | Custo de acesso |
|--------|-------|------------------------|-----------------|
| **Hot** (quente) | Acessados com frequência | Mais alto | Mais baixo |
| **Cool** (fria) | Acessados raramente (mín. ~30 dias) | Mais baixo | Mais alto |
| **Cold** (gelada) | Acessados muito raramente (mín. ~90 dias) | Ainda menor | Ainda maior |
| **Archive** (arquivo morto) | Quase nunca acessados (mín. ~180 dias) | O mais baixo | O mais alto; exige **reidratação** |

> A camada **Archive** é offline: para ler os dados é preciso "reidratá-los" para Hot ou Cool, o que leva tempo.

---

## Gerenciamento de Ciclo de Vida

O Blob Storage permite **regras de ciclo de vida** que movem blobs automaticamente entre camadas (ex.: de Hot para Cool após 30 dias) ou os excluem após certo tempo — reduzindo custos sem intervenção manual.

---

## Casos de Uso

- Servir imagens e vídeos para sites e aplicativos.
- Armazenar arquivos para backup e recuperação de desastres.
- Guardar dados brutos para análise (base de **data lakes**).
- Armazenar logs e dados de arquivamento.

> O **Azure Data Lake Storage Gen2** é construído sobre o Blob Storage, adicionando um **namespace hierárquico** (pastas reais) para cargas analíticas de big data.

---

## Pontos-Chave para o Exame

- ✅ Blob Storage guarda **dados não estruturados** como objetos.
- ✅ Hierarquia: **conta → contêiner → blob**.
- ✅ Camadas: **Hot, Cool, Cold, Archive** — escolha pelo equilíbrio custo × frequência de acesso.
- ✅ **Archive** é a mais barata para guardar, mas exige **reidratação** para leitura.
- ✅ **Data Lake Storage Gen2** = Blob Storage com namespace hierárquico para analytics.
