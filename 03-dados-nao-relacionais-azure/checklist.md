# Checklist de Revisão — Módulo 3 (Dados Não Relacionais no Azure)

## 3.1 Armazenamento do Azure — Visão Geral
- [ ] Sei que a **conta de armazenamento** agrupa Blob, Files, Table e Queue
- [ ] Lembro que o foco do DP-900 é **Blob, Files e Table**
- [ ] Sei que os dados são **criptografados em repouso** e replicados
- [ ] Diferencio **LRS, ZRS, GRS e GZRS**

## 3.2 Azure Blob Storage
- [ ] Sei que o Blob guarda **dados não estruturados** como objetos
- [ ] Conheço a hierarquia **conta → contêiner → blob**
- [ ] Diferencio os tipos de blob (**block, append, page**)
- [ ] Conheço as camadas **Hot, Cool, Cold, Archive** e seu trade-off de custo
- [ ] Sei que **Archive** exige **reidratação** para leitura
- [ ] Sei o que é o **Data Lake Storage Gen2**

## 3.3 Azure Files e Table Storage
- [ ] Sei que o **Azure Files** usa **SMB/NFS** e é montável como unidade de rede
- [ ] Sei que o **Table Storage** é NoSQL de **chave-valor**, sem esquema
- [ ] Entendo o papel de **Partition Key** e **Row Key**

## 3.4 Azure Cosmos DB — Visão Geral
- [ ] Descrevo o Cosmos DB como NoSQL **distribuído globalmente** e de baixa latência
- [ ] Sei que o desempenho é medido em **Request Units (RUs)**
- [ ] Conheço os modos de throughput (provisionado, serverless, autoscale)
- [ ] Sei dar **casos de uso** (IoT, varejo global, web/mobile, jogos)

## 3.5 APIs do Cosmos DB
- [ ] Conheço as **5 APIs**: NoSQL, MongoDB, Cassandra, Gremlin, Table
- [ ] Sei que a **API for NoSQL** é a nativa, para projetos novos
- [ ] Sei que **Gremlin** é para grafos
- [ ] Entendo que MongoDB/Cassandra/Table servem à **compatibilidade/migração**
