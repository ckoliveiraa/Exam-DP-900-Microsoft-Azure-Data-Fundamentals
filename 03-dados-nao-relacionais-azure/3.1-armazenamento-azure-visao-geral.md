# 3.1 — Armazenamento do Azure: Visão Geral

> O **Armazenamento do Azure** (*Azure Storage*) é o serviço gerenciado da Microsoft para guardar dados **não relacionais** na nuvem — de arquivos e imagens a tabelas e filas de mensagens.

---

## A Conta de Armazenamento

Tudo começa por uma **conta de armazenamento** (*storage account*) — o contêiner de nível superior que agrupa os serviços de dados e define configurações como localização, redundância e desempenho.

Dentro de uma conta de armazenamento existem **quatro serviços**:

| Serviço | Para que serve |
|---------|----------------|
| **Blob Storage** | Objetos/arquivos não estruturados em larga escala |
| **Azure Files** | Compartilhamentos de arquivos acessados pela rede |
| **Table Storage** | Dados NoSQL de chave-valor |
| **Queue Storage** | Filas de mensagens entre componentes de uma aplicação |

> 📌 No exame **DP-900**, o foco recai sobre **Blob**, **Files** e **Table**. O Queue Storage aparece apenas como serviço da conta de armazenamento.

---

## Características do Armazenamento do Azure

- **Durável e altamente disponível** — os dados são replicados automaticamente.
- **Seguro** — criptografia automática dos dados em repouso.
- **Escalável** — suporta volumes massivos de dados.
- **Gerenciado** — a Microsoft cuida de hardware, manutenção e correções.
- **Acessível** — via HTTP/HTTPS, SDKs, CLI, portal e APIs REST.

---

## Regiões e Zonas de Disponibilidade

Antes de falar de redundância, é preciso entender **onde** os dados podem ser replicados:

- **Região** — uma área geográfica do Azure (ex.: *Brazil South*, *East US*) que contém um ou mais datacenters.
- **Zona de disponibilidade** — datacenters fisicamente separados **dentro da mesma região**, com energia, refrigeração e rede independentes. Protegem contra falhas localizadas.
- **Região secundária (par regional)** — outra região, **geograficamente distante**, usada para recuperação de desastres.

![Regiões e Zonas de Disponibilidade no Azure](https://learn.microsoft.com/pt-br/azure/reliability/media/availability-zones-overview/regions-availability-zones.svg)

> 💡 Cada nível (datacenter → zona → região) aumenta a proteção contra falhas — e é essa hierarquia que define as opções de redundância a seguir.

---

## Redundância (Replicação)

Os dados são sempre replicados para resistir a falhas. As opções aumentam a proteção:

| Opção | Replicação |
|-------|-----------|
| **LRS** (*Locally Redundant Storage*) | 3 cópias dentro de **um único datacenter**. |
| **ZRS** (*Zone-Redundant Storage*) | Cópias em **zonas de disponibilidade** diferentes da mesma região. |
| **GRS** (*Geo-Redundant Storage*) | LRS + cópia em uma **região secundária** (geograficamente distante). |
| **GZRS** (*Geo-Zone-Redundant Storage*) | ZRS + cópia em uma região secundária. |

> Quanto maior a redundância, maior a proteção — e o custo.

---

## Pontos-Chave para o Exame

- ✅ A **conta de armazenamento** agrupa os serviços: **Blob, Files, Table e Queue**.
- ✅ O foco do DP-900 está em **Blob, Files e Table** (dados não relacionais).
- ✅ Os dados são **criptografados em repouso** e **replicados** automaticamente.
- ✅ **LRS** = um datacenter; **GRS** = inclui região secundária para recuperação de desastres.
