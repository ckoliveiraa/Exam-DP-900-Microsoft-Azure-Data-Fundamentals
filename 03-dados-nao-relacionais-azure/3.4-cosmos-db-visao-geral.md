# 3.4 — Azure Cosmos DB: Visão Geral e Casos de Uso

> O **Azure Cosmos DB** é o banco de dados **NoSQL distribuído globalmente** e **totalmente gerenciado** da Microsoft, projetado para aplicações modernas que exigem **baixíssima latência** e **escala elástica** em qualquer lugar do mundo.

---

## Características Principais

- **Distribuição global** — os dados podem ser replicados para **qualquer região** do Azure com poucos cliques.
- **Baixa latência** — respostas em milissegundos, garantidas por **SLA**.
- **Escala elástica** — ajusta automaticamente capacidade de processamento e armazenamento.
- **Multimodelo** — suporta vários modelos de dados por meio de diferentes **APIs** (ver aula 3.5).
- **Totalmente gerenciado** — sem servidores, patches ou backups manuais.
- **Alta disponibilidade** — SLA de até 99,999% com gravações multirregião.

---

## Unidades de Requisição (RUs)

O desempenho do Cosmos DB é medido em **Request Units (RUs)** — uma unidade que representa o custo de processamento de uma operação (ler, gravar, consultar).

| Modo de throughput | Como funciona |
|--------------------|---------------|
| **Provisionado** | Você reserva uma quantidade de RUs/s; bom para cargas previsíveis. |
| **Serverless** | Paga apenas pelas RUs consumidas; bom para cargas intermitentes. |
| **Autoscale** | Ajusta as RUs automaticamente conforme a demanda. |

> O Cosmos DB tem um **nível gratuito** que oferece uma cota de RUs/s e de armazenamento sem custo.

---

## Quando Usar o Cosmos DB

É indicado para cenários de **alta escala**, **distribuição global** e **resposta rápida**:

| Cenário | Por quê |
|---------|---------|
| **IoT e telemetria** | Ingestão massiva e contínua de dados de dispositivos |
| **Varejo e e-commerce** | Catálogos globais e picos de tráfego (ex.: Black Friday) |
| **Aplicações web e mobile** | Baixa latência para usuários no mundo todo |
| **Jogos** | Placar e estado de jogo com resposta instantânea em escala |

---

## Cosmos DB × Bancos Relacionais

| | Cosmos DB (NoSQL) | Banco relacional |
|---|---|---|
| Esquema | Flexível | Fixo |
| Distribuição global | Nativa, com poucos cliques | Complexa |
| Escala | Horizontal e elástica | Principalmente vertical |
| Latência | Milissegundos garantidos por SLA | Variável |
| Ideal para | Apps modernas, globais, alta escala | Dados estruturados com integridade forte |

---

## Pontos-Chave para o Exame

- ✅ Cosmos DB = NoSQL **distribuído globalmente**, gerenciado, de **baixa latência**.
- ✅ Replica dados para **várias regiões** facilmente; SLA de alta disponibilidade.
- ✅ O desempenho é medido em **Request Units (RUs)**.
- ✅ Casos típicos: **IoT, varejo global, apps web/mobile, jogos**.
- ✅ É **multimodelo** — suporta diferentes APIs (próxima aula).
